Recommended use:

- Use `.gif` or `.webp` for short inline README before/after clips.
- Use `.mp4` for higher-quality demos that should be linked from the README or attached to a GitHub Release.
- Keep filenames descriptive and stable, for example:
  - `timeline-dense-to-sparse.gif`
  - `motion-path-smoothing-before-after.mp4`
  - `shape-path-vertex-reduction.webp`

Keep source footage license-clean and avoid project-internal branding unless it
is meant to appear in the public `bbsolver` repository.
